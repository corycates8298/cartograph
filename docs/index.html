🧭 Cartograph

v1.0 · MIT 33/33 tests passing

A customer purchase analytics engine that maps the terrain of buyer behavior using vector embeddings and a conversational AI agent.

“If a guest purchased body lotion between January 3, 2025 and January 2, 2026, how many also purchased laundry soap in a similar fragrance?”

The word similar is what kills ThoughtSpot, Fabric, and Tableau. Cartograph solves it with scent-profile vector embeddings — Lavender matches Hey Headache (both calming/herbal), Grapefruit Mimosa matches Lemon Drop (both tart citrus) — not because of SQL JOINs but because of cosine similarity in embedding space.

Fragrance taxonomy

51 fragrances mapped to scent note profiles across 8 families. The embedding engine uses these note profiles (not names) to compute similarity — so “Grapefruit Mimosa” correctly matches “Lemon Drop” (0.715 similarity) even though they share zero words.

Why it exists

Standard BI tools (ThoughtSpot, Tableau, Fabric) can handle cohort filtering and cross-category joins. They cannot handle semantic matching — “similar fragrance” isn't a JOIN key. Cartograph solves this by embedding scent profiles into vector space, making similarity a computable function rather than a manual taxonomy. Same approach used by Symrise/IBM's Philyra system for commercial fragrance development, applied here to retail cross-sell analytics.
